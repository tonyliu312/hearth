# L1 Baseline · F01=25 / F02=10  ·  2026-06-02 01:16 CST

Captured immediately after changing the AC built-in controller setpoint
back to a more permissive configuration, before any L2 (Hearth controller)
is enabled. This is the **24h-comparison reference point** for verifying
whether L1 alone settles into a reasonable duty cycle and whether L2
(when later enabled) actually does better.

---

## L1 settings (in the AC LCD panel)

| Param | Value | Meaning |
|---|---|---|
| **F01** 制冷停止温度 | **25 °C** | AC stops cooling when return-air ≤ 25 °C |
| **F02** 制冷回差 | **10 °C** | Hysteresis — AC starts cooling when return-air ≥ F01+F02 = **35 °C** |
| **F05** 柜内高温告警 | **25 °C** | Alarm threshold (capped at F01 by firmware) |
| **F10** 内风机模式 | **on** (long-run) | Internal fan always on — promotes air mixing |

## Why this setpoint

Previous attempt (F01=24 / F02=10 / F05=24) ran the AC at near-100% duty
because the start temperature (F01+F02 = 27 °C) was too close to the
cabinet's natural ambient (~30 °C with sun load). AC could never let
return-air rise above the start threshold long enough to actually stop.

New setpoint moves start temperature to 35 °C, giving the AC headroom to
genuinely cycle off. Stop temperature 25 °C means deep cooling per cycle
but longer off-time between cycles → fewer compressor starts, lower
total duty.

## Next step (after this baseline is validated)

If L1 alone settles into 30–60% duty and GPU stays healthy → **good
baseline established**. Then enable L2 (Hearth controller) to opportunistically
turn off the plug during periods when GPUs are cool, expecting an
additional 10–25% reduction without compromising safety.

L2 design guarantees:
1. L1 keeps full control whenever L2 chooses to keep the plug on.
2. L2 can ONLY turn off (override L1's "want to cool"); it cannot
   prevent L1 from cooling when L2 has plug on.
3. Three-layer watchdog ensures L2 failure → plug back on → L1 takes
   over within minutes.

See `docs/topology.md` `ha.controller` section once that lands.
